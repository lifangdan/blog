    BFC详解

    非洲猫 11/20/2021 CSS

    # 1、什么是BFC

    BFC(Block formatting context)直译为"块级格式化上下文"。它是一个独立的渲染区域。一个BFC区域包含创建该上下文元素的所有子元素，但是不包括创建了新的BFC的子元素的内部元素。

    每一个BFC区域只包括其子元素，不包括其子元素的子元素。
    每一个BFC区域都是独立隔绝的，互不影响！

    # 2、触发BFC的条件

    并不是任意一个元素都可以被当做BFC，只有当这个元素满足以下任意一个条件的时候，这个元素才会被当做一个BFC。

    • body根元素
    • 设置浮动，不包括none
    • 设置定位，absoulte或者fixed
    • 行内块显示模式，inline-block
    • 设置overflow，即hidden，auto，scroll
    • 表格单元格，table-cell
    • 弹性布局，flex

    # 3、BFC布局与普通文档流布局区别

    # 普通文档流布局规则

    • 两个相邻元素上下margin会重叠
    • margin会传递给父级
    • 浮动的元素是不会被父级计算高度
    • 浮动元素会覆盖在非浮动元素上

    # BFC布局规则

    • 两个相邻元素上下margin不会重叠（给其中一个元素增加一个父级，然后让他的父级触发BFC）
    • margin不会传递给父级（父级触发了BFC）
    • 浮动的元素会被父级计算高度（父级触发了BFC）
    • 浮动元素不会覆盖在非浮动元素上（非浮动元素触发了BFC）

    # 4、BFC的作用

		width: 100px;
		height: 100px;
		background: green;
		margin-top: 50px;
	}

	.bfc {
		overflow: hidden;
	}
</style>

    # 解决子元素的margin值传递给父级的问题

    同理，也可以利用BFC解决包含塌陷问题，即当父子关系的盒子，给子元素添加margin值，父元素也会被影响。此时给父元素也加上overflow: hidden触发BFC,则子元素的margin不会传递给父级。

    没有触发BFC时：

    <template>
	<div>
		<div class="bfc">
			<div class="div1"></div>
		</div>
	</div>
</template>
<style>
	.div1 {
		width: 100px;
		height: 100px;
		background: red;
		margin-top: 50px;
	}
	.bfc {
		background: green;

    # 解决浮动元素覆盖在非浮动元素上的问题(自适应两栏布局)

    红色盒子向左浮动，绿色盒子是标准流，默认情况下，浮动元素覆盖在标准流元素上。

    <template>
	<div>
		<div class="left"></div>
		<div class="right"></div>
	</div>
</template>
<style>
	.left {
		width: 100px;
		height: 100px;
		background: red;
		float: left;
	}

	.right {
		background: green;
		height: 200px;
	}
</style>
