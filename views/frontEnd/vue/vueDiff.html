    Vue踩过的坑

    非洲猫 9/4/2019 Vue

    # 项目部署测试服务器，刷新子页面出现404问题

    路由模式选择history模式的时候，刷新子页面，服务器没有响应的资源，会刷出404。

    const router = new Router({
mode: 'history'
});
    1
    2
    3

    解决方案：把history模式换成hash模式，hash模式url会多出一个#。如果依然使用history模式，那就需要后台的配合修改，需要把所有路由都重定向到根页面。

    # vue-router的beforeEach第一次打开页面不执行

    vue.use的时候已经初始化了，挂载的时候hash值没有发生变化，所以不会执行beforeEach

    所以正确的代码顺序如下（先路由再初始化）：

    router.beforeEach((to, from, next) => {
  //token校验
});

new Vue({
  router,
  store,
  render: (h) => h(App),
}).$mount("#app");
